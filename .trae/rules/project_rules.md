# 项目规则

## 修改前必读
- `personal_rules.md` - 工作规则
- `feature_list.md` - 功能依赖
- `change_checklist.md` - 检查清单

## 修改前4问
1. 改什么功能？
2. 依赖哪些功能？
3. 哪些功能依赖它？
4. 需要测试哪些？

## 修改规则
✅ 一次只改一个地方
✅ 保留原始代码
❌ 禁止同时改多个功能
❌ 禁止跳过测试

## 修改后必测
- [ ] 应用正常启动
- [ ] 被改功能正常
- [ ] 相关功能正常
- [ ] WebSocket/去重/弹窗/复制/托盘
- [ ] 控制台无错误
- [ ] 重启后正常

## 危险操作
- 修改全局变量名
- 修改事件名称
- 修改函数签名
- 修改数据结构

## 记住
**测试不充分 = 一定会出问题！**
