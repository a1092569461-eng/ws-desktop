# 代码变更检查清单 - 精简版

## 修改前（必须做）

1. **明确目标** - 我要修改什么功能？为什么？要达到什么效果？
2. **查看清单** - 打开 `feature_list.md`，找到对应功能
3. **分析影响** - 依赖哪些功能？哪些功能依赖它？
4. **制定计划** - 列出修改位置、顺序、需要测试的功能

## 修改中（谨慎操作）

1. **备份代码** - 确保可以回滚
2. **最小化修改** - 一次只改一个功能
3. **保留兼容** - 先注释旧代码，确认正常后再删除
4. **代码审查** - 检查语法、逻辑、调用点

## 修改后（必须测试）

### 阶段1：基础测试
- [ ] 应用能正常启动，无编译/运行时错误
- [ ] 被修改的功能正常工作
- [ ] 相关功能没有被破坏

### 阶段2：完整测试
- [ ] WebSocket 连接/接收消息
- [ ] 消息去重功能
- [ ] 重点消息检测+弹窗
- [ ] 复制功能
- [ ] 窗口置顶+状态记忆
- [ ] 系统托盘

### 阶段3：深度检查
- [ ] 打开开发者工具，无红色错误
- [ ] 查看Rust日志，无异常
- [ ] 长时间运行，内存稳定

### 阶段4：最终验证
- [ ] 关闭应用，重新打开
- [ ] 应用能正常启动
- [ ] 之前的设置被正确保存
- [ ] 所有功能仍然正常

## 发现问题怎么办

1. 立即停止修改
2. 恢复到修改前的状态
3. 重新分析问题
4. 逐步修改，逐步测试

## 绝对禁止

- ❌ 不要在没有理解代码的情况下修改
- ❌ 不要同时修改多个不相关的功能
- ❌ 不要删除不确定是否使用的代码
- ❌ 不要跳过测试步骤
- ❌ 不要说"应该没问题"而不测试

## 快速检查表（小修改用）

- [ ] 应用能正常启动
- [ ] 被修改的功能正常
- [ ] 控制台没有错误
- [ ] 关闭重启后仍然正常

**注意：核心功能修改必须用完整检查清单！**

## 记住

**宁可多花时间测试，也不要花时间修复bug！**
**测试不充分 = 一定会出问题！**
